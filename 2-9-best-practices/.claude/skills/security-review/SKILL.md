---
name: security-review
description: 对当前代码库进行安全审查，检查 OWASP Top 10 常见漏洞
model: opus
---

对指定文件或目录执行安全审查，重点检查以下问题：

1. **注入攻击**：SQL 注入、命令注入、XSS（检查未转义的用户输入）
2. **认证与授权**：硬编码凭证、弱密码策略、缺少权限校验
3. **敏感数据暴露**：日志中输出密码/token、响应体包含敏感字段
4. **依赖安全**：已知漏洞的依赖版本（检查 package.json）

输出格式：

```
## 安全审查报告

### 🔴 高危（需立即修复）
- [文件:行号] 问题描述 + 修复建议

### 🟡 中危（本周内修复）
- [文件:行号] 问题描述 + 修复建议

### 🟢 低危（下次迭代修复）
- [文件:行号] 问题描述 + 修复建议
```

没有问题时明确说"未发现 XXX 类问题"，不要省略分类。
